What an ERP.net App is

An ERP.net App is any application, integration, or service that connects to an ERP.net instance to extend, customize, or interact with its business data and functionality. Apps operate outside (or alongside) the instance, communicate through the instance's public APIs, and are recognized by the system through a Trusted Application. Authentication and authorization are handled through the built-in ERP.net Identity.

See: What is an ERP.net App

Application types

ERP.net supports several application types, depending on where they run, who develops them, and how they are distributed. All types follow the same core security/connectivity model (Identity Server + APIs + Trusted Application representation), but differ in deployment and distribution:

  • Internal applications – part of the ERP.net platform (e.g., Web Client, Client Center).
  • External applications – built outside ERP.net (customer/partner/third-party) and registered per instance.
  • Marketplace applications – external apps distributed via ERP.net Marketplace and auto-registered on installation.
  • Embedded (in-client) applications – external web apps displayed inside the Web Client (with special behavior depending on hosting domain and trust).

See: Application Types

How apps connect

Apps connect through standard OAuth 2.0 / OIDC mechanisms and must choose the correct identity authority depending on the scenario (instance API access vs external SSO).

API access options

ERP.net provides more than one API surface. The choice affects how you model data, how you query, and what kinds of operations are available.

See: API Access (Domain API vs Table API).

Calling APIs safely

A secure integration is not only about choosing the right OAuth flow-your app must also handle tokens correctly, protect secrets, and apply least-privilege access.
